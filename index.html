Questionário

Questão 1: Explique qual é a função de um servidor web em um ambiente de hospedagem e cite pelo menos um exemplo.

Resposta: Um servidor web armazena e entrega arquivos de um site (HTML, CSS, imagens) para o navegador do usuário. Um exemplo é o Apache HTTP Server.
Questão 2: Qual a principal característica do NGINX e por que ele é muito utilizado em projetos de alto desempenho?

Resposta: A principal característica do NGINX é sua arquitetura orientada a eventos, que o torna muito eficiente em lidar com muitas conexões simultâneas. Por isso, é muito usado em projetos de alto desempenho, agindo como proxy reverso e balanceador de carga.
Questão 3: Diferencie os protocolos HTTP e HTTPS. Por que o HTTPS é recomendado para aplicações modernas?

Resposta: HTTP é um protocolo para transferir dados na web sem criptografia. HTTPS é a versão segura do HTTP, que criptografa a comunicação. O HTTPS é recomendado para aplicações modernas porque garante a privacidade e a integridade dos dados, protegendo contra interceptações e adulterações.
Questão 4: Qual é a função das portas em uma rede de computadores e qual a importância da porta 443?

Resposta: As portas em uma rede de computadores identificam serviços específicos em um dispositivo. A porta 443 é importante porque é a porta padrão para o tráfego HTTPS, ou seja, para comunicação web segura.
Questão 5: Defina o conceito de endereço IP e explique sua importância em redes de computadores.

Resposta: Um endereço IP é um número único que identifica um dispositivo em uma rede de computadores. É crucial para que os dispositivos possam se comunicar entre si, permitindo que os dados sejam enviados e recebidos no destino correto.
Questão 6: Explique o que é hospedagem VPS e em quais situações ela é mais indicada.

Resposta: Hospedagem VPS é um servidor virtualizado que oferece recursos dedicados e mais controle do que a hospedagem compartilhada. É mais indicada para sites e aplicações que precisam de mais recursos, segurança e personalização, mas não justificam um servidor dedicado.
Questão 7: Descreva as vantagens do servidor web Caddy em relação a outros servidores tradicionais.

Resposta: As vantagens do servidor web Caddy incluem sua configuração simples com arquivos de texto, a geração e renovação automática de certificados HTTPS (Let's Encrypt), e sua capacidade de agir como proxy reverso e balanceador de carga de forma nativa.
Questão 8: Qual servidor de aplicação é mais indicado para aplicações desenvolvidas na plataforma .NET (C#)? Explique o motivo.

Resposta:O servidor de aplicação mais indicado para aplicações desenvolvidas na plataforma .NET (C#) é o Microsoft IIS (Internet Information Services). O motivo é que ele é um produto da Microsoft, otimizado para o ecossistema .NET, oferecendo integração perfeita e desempenho superior para essas aplicações.
Questão 9: Descreva o funcionamento do modelo Cliente-Servidor no contexto de sistemas web.

Resposta:No modelo Cliente-Servidor em sistemas web, o cliente (o navegador do usuário) faz uma requisição por uma página ou dado ao servidor (onde o site está hospedado). O servidor processa a requisição e envia a resposta de volta ao cliente.
Questão 10: Explique o que é o Apache Tomcat e para qual tipo de aplicação ele é mais indicado.

Resposta:O Apache Tomcat é um servidor de aplicações web de código aberto que implementa as especificações Java Servlet, JavaServer Pages (JSP), Java Expression Language e Java WebSocket. É mais indicado para aplicações web desenvolvidas em Java.
Questão 11: Diferencie, com suas palavras, um servidor web de um servidor de aplicação, destacando suas funções principais.

Resposta:O Um servidor web serve arquivos estáticos (HTML, CSS, imagens) e lida com requisições HTTP básicas. Um servidor de aplicação, por outro lado, executa código dinâmico de aplicações (como Java, Python, .NET), processa lógica de negócio e interage com bancos de dados, gerando conteúdo dinâmico antes de enviá-lo ao servidor web ou diretamente ao cliente.
Questão 12: Defina o conceito de domínio na internet e sua relação com os endereços IP.

Resposta:Um domínio na internet é um nome legível por humanos (ex: https://www.google.com/search?q=google.com) usado para identificar sites e serviços online. Ele se relaciona com o endereço IP porque funciona como um "apelido" ou "atalho" para o endereço IP numérico, facilitando a memorização e o acesso aos sites, já que o DNS (Domain Name System) traduz o domínio para o IP correspondente.
Questão 13: Quais as principais vantagens da hospedagem em nuvem em comparação com a hospedagem local?

Resposta:As principais vantagens da hospedagem em nuvem em comparação com a hospedagem local são: escalabilidade (facilidade de aumentar ou diminuir recursos), alta disponibilidade (menos tempo de inatividade), flexibilidade, redução de custos de infraestrutura e segurança (muitas vezes gerenciada pelo provedor).
Questão 14:Cite dois exemplos de servidores de aplicação e as linguagens/plataformas que eles suportam.

Resposta:Apache Tomcat: Suporta Java. Microsoft IIS: Suporta .NET (C#, VB.NET).
Questão 15:Por que é essencial a utilização de HTTPS em sites e sistemas atualmente? Quais riscos o HTTP apresenta?

Resposta:A utilização de HTTPS é essencial atualmente para garantir a segurança e a privacidade dos dados dos usuários. O HTTP apresenta riscos como: interceptação de dados (escuta de informações sensíveis), adulteração de dados (modificação de informações em trânsito) e falta de autenticidade (não garante que você está se comunicando com o servidor correto).
Questão 16:Dê um exemplo de situação onde o uso de hospedagem VPS seria mais vantajoso do que uma hospedagem compartilhada.

Resposta:O uso de hospedagem VPS seria mais vantajoso do que uma hospedagem compartilhada para um e-commerce de médio porte com um tráfego crescente. Em uma hospedagem compartilhada, picos de acesso ou recursos consumidos por outros sites no mesmo servidor poderiam causar lentidão ou queda, enquanto o VPS garantiria recursos dedicados e melhor desempenho para o e-commerce.
Questão 17:Diferencie os conceitos de IP e porta em uma rede, explicando suas funções.

Resposta:IP é o endereço numérico que identifica um dispositivo na rede (onde o dado deve ir). A porta é um número que identifica um serviço específico dentro daquele dispositivo (para qual aplicação o dado deve ir).
Questão 18:Qual a importância de um servidor web conseguir lidar com conexões via HTTPS?

Resposta:É importante que um servidor web consiga lidar com conexões via HTTPS para proteger a comunicação entre o usuário e o site, garantir a privacidade dos dados trocados (senhas, informações bancárias), e estabelecer confiança através de certificados digitais, que são cruciais para a segurança online
Questão 19:Cite dois exemplos de servidores web e explique suas principais características.

Resposta:Apache HTTP Server: Servidor web mais antigo e amplamente utilizado, conhecido por sua flexibilidade e grande comunidade. NGINX: Conhecido por sua alta performance, eficiência em lidar com muitas conexões simultâneas e por ser frequentemente usado como proxy reverso e balanceador de carga.
Questão 20:Defina o que é uma hospedagem compartilhada e em quais casos ela costuma ser utilizada.

Resposta:Hospedagem compartilhada é um tipo de hospedagem onde vários sites compartilham os recursos de um único servidor. Ela costuma ser utilizada em casos de sites pequenos, blogs pessoais ou projetos iniciantes com baixo volume de tráfego, pois é a opção mais econômica e simples de gerenciar.
Questão 21:Explique a função de um servidor de aplicação e a diferença em relação a servidores que apenas servem arquivos estáticos.

Resposta:Um servidor de aplicação executa o código dinâmico da aplicação, processa a lógica de negócio, interage com bancos de dados e gera conteúdo personalizado. A diferença para servidores que apenas servem arquivos estáticos (como um servidor web simples) é que estes últimos só entregam arquivos pré-existentes, sem processamento de código ou lógica de negócio.
Questão 22:Cite um servidor de aplicação usado para projetos em Python e descreva como ele funciona.

Resposta:Um servidor de aplicação usado para projetos em Python é o Gunicorn (Green Unicorn). Ele funciona como um WSGI HTTP Server, ou seja, ele recebe as requisições HTTP do cliente e as encaminha para a aplicação Python (escrita usando frameworks como Flask ou Django) através da interface WSGI (Web Server Gateway Interface), e depois envia a resposta de volta ao cliente.
Questão 23: O que é o WildFly (ex-JBoss) e para qual tecnologia ele é indicado?

Resposta:O WildFly (anteriormente JBoss Application Server) é um servidor de aplicação de código aberto que implementa as especificações Java EE (Enterprise Edition). Ele é indicado para aplicações corporativas complexas e de grande escala desenvolvidas em Java EE, oferecendo um ambiente robusto e completo para essas aplicações.
Questão 24: Qual a função da porta 21 em redes de computadores e para qual serviço ela é utilizada?

Resposta:A porta 21 em redes de computadores é utilizada para o serviço FTP (File Transfer Protocol). Sua função é controlar a conexão para a transferência de arquivos entre um cliente e um servidor.
Questão 25: Explique por que o servidor LiteSpeed é compatível com o Apache e quais as vantagens dessa compatibilidade.

Resposta:O servidor LiteSpeed é compatível com o Apache porque ele pode ler e usar os arquivos de configuração do Apache (.htaccess). As vantagens dessa compatibilidade incluem a facilidade de migração (não é preciso reconfigurar tudo), melhor desempenho (LiteSpeed é geralmente mais rápido que o Apache), e otimização de recursos.
Questão 26: O que é o Microsoft IIS e em quais situações ele costuma ser adotado?

Resposta:O Microsoft IIS (Internet Information Services) é um servidor web e de aplicação desenvolvido pela Microsoft para sistemas Windows. Ele costuma ser adotado em situações onde há aplicações desenvolvidas na plataforma .NET, ambientes Windows Server, e em empresas que já utilizam outras tecnologias Microsoft em sua infraestrutura.
Questão 27: Explique para que serve o Node.js e em quais tipos de aplicações ele é mais utilizado.

Resposta:Node.js é um ambiente de tempo de execução (runtime) JavaScript de código aberto que permite executar JavaScript no lado do servidor. Ele serve para construir aplicações de rede escaláveis e de alta performance. É mais utilizado em tipos de aplicações como APIs RESTful, microsserviços, aplicações de tempo real (chats, jogos online) e Single Page Applications (SPAs) com backend JavaScript.
Questão 28: Descreva a função do servidor GlassFish e para qual linguagem ele é destinado.

Resposta:O servidor GlassFish é um servidor de aplicação de código aberto que implementa as especificações Java EE. Sua função é hospedar e executar aplicações Java EE. Ele é destinado a projetos desenvolvidos na linguagem Java.
Questão 29: O que são Puma e Unicorn e em quais projetos eles são normalmente utilizados?

Resposta:Puma e Unicorn são servidores HTTP que servem aplicações Ruby. Eles são normalmente utilizados em projetos Ruby on Rails e outras aplicações Ruby, atuando como interfaces entre o servidor web (como NGINX) e a aplicação Ruby em si, gerenciando os processos e threads da aplicação.
Questão 30: Qual a importância da escolha adequada entre hospedagem local, compartilhada, VPS e nuvem no planejamento de um projeto web?

Resposta:A escolha adequada entre hospedagem local, compartilhada, VPS e nuvem é crucial no planejamento de um projeto web porque afeta diretamente o desempenho, escalabilidade, segurança, custo e gerenciamento do site ou aplicação. Uma escolha errada pode levar a problemas de lentidão, custos desnecessários, falta de recursos ou vulnerabilidades de segurança, impactando o sucesso do projeto.
